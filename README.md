# HTML-Practice-Assignment-One

This project is a practice assignment focused on HTML and CSS. It is designed to help learn and build foundational web development skills by working with basic HTML structure, styling with CSS, and organizing assets such as images and fonts. The repository includes example files and folders for hands-on practice.

## Branching Strategy

- **main**: The main branch contains the stable, production-ready code. All completed and reviewed work is merged here.
- **feature branches**: For each new feature or task, create a separate branch from `main` (e.g., `feature/navbar`, `feature/footer`).
- **bugfix branches**: For bug fixes, create a branch from `main` (e.g., `bugfix/image-path`).
- **Pull Requests**: All changes should be merged into `main` via pull requests. Code reviews are encouraged before merging.

This strategy helps keep the codebase organized and ensures that the main branch always contains working code.

Since the project was from scratch commits of placeholder files, such as .gitkeep, .gitignore, and readme, were made directly to main rather than with a separate branch as there was no risk of harm. Updates to these files or their removal were also committed directly to main for the same reason.

The commits were inline with what I would normally do with my current team but broken up slightly differently. A single commit typically being all the changes needed to a single file rather than broken up into multiple commits and pull requests. This was done more for practice than anything else.

In production I would most likely combine some of the commits to be larger due to the changes being low complexity.
